# AGENTS.md

## 커밋 규칙

- 커밋은 작게 만들고, 하나의 의도만 담는다.
- 관련 없는 작업 트리 변경사항을 커밋에 포함하지 않는다.
- 커밋하기 전 실제 `git status`를 확인한다.
- 서로 관련 없는 변경사항이 여러 개 있으면 커밋을 분리한다.
- 사용자가 명시적으로 요청하지 않는 한 기존 커밋을 amend 하거나 rewrite 하지 않는다.
- 사용자가 명시적으로 요청하지 않는 한 사용자의 변경사항을 되돌리지 않는다.
- 사용자가 영어를 요청하지 않는 한 커밋 메시지는 한국어로 작성한다.
- 커밋 메시지는 아래 형식을 우선 사용한다.

```text
type: 짧은 요약

변경한 내용:
- 실제로 수정한 파일, 기능, 설정을 구체적으로 적는다.
- 단순히 "수정함"처럼 쓰지 말고 무엇이 어떻게 바뀌었는지 적는다.

변경한 이유:
- 왜 이 변경이 필요한지 적는다.
- 버그, 요구사항, 유지보수, 보안, 설정 정리 등 변경의 배경을 분명히 적는다.

검증:
- 실제로 실행한 명령만 적는다.
- 실행하지 않은 테스트, 린트, 타입체크, 빌드는 적지 않는다.
- 실패한 명령이 있다면 실패 사실과 핵심 에러를 함께 적는다.
```

## 커밋 분리 기준

- 하나의 커밋에 모든 변경사항을 몰아넣지 않는다.
- diff의 성격이 다르면 커밋을 분리한다.
- 기능 추가, 버그 수정, 리팩터링, 문서 변경, 설정 변경, 의존성 변경은 가능한 한 별도 커밋으로 나눈다.
- 같은 파일을 수정했더라도 변경 의도가 다르면 커밋을 분리한다.
- 여러 파일을 수정했더라도 하나의 목적을 위해 함께 바뀐 경우에는 하나의 커밋으로 묶을 수 있다.
- 커밋 단위는 나중에 되돌리거나 리뷰할 때 의미가 분명해야 한다.
- 커밋을 만들기 전 `git diff`를 보고 변경사항을 성격별로 나눌 수 있는지 먼저 판단한다.

## 검증 규칙

- 매 작업마다 습관적으로 테스트, 린트, 타입체크, 빌드를 모두 실행하지 않아도 된다.
- 변경 범위가 작고 문서, 설정 설명, 주석처럼 런타임 동작에 영향이 없는 경우에는 불필요한 검증을 생략할 수 있다.
- 검증은 변경의 위험도와 성격에 맞게 선택한다.
- 코드 동작, 타입, 빌드 결과에 영향을 줄 수 있는 변경이면 필요한 검증을 선별해서 실행한다.
- 시간이 오래 걸리거나 범위가 큰 검증이 필요하면 먼저 사용자에게 실행 여부를 확인한다.
- 검증을 생략했다면 커밋 본문이나 작업 요약에 실행하지 않은 검증을 실행한 것처럼 쓰지 않는다.

- 커밋 타입은 아래 기준을 우선 사용한다.

  - `feat`: 사용자에게 보이는 기능 추가
  - `fix`: 버그 수정
  - `refactor`: 동작을 바꾸지 않는 코드 정리
  - `chore`: 도구, 설정, 의존성, 유지보수 변경
  - `docs`: 문서만 변경
  - `style`: 포맷팅만 변경
  - `test`: 테스트만 변경

- 현재 세션에서 직접 실행하지 않은 테스트, 린트, 타입체크, 빌드는 통과했다고 쓰지 않는다.
- 반복적이고 가치가 낮은 검증 내용은 커밋 본문에 넣지 않는다.
- 명령이 실패하면 숨기지 말고 실패한 정확한 명령과 핵심 에러를 적는다.

## 보안 감사 규칙

- 사용자가 `/cso`를 요청하면 `.agents/skills/gstack-cso/SKILL.md`의 Chief Security Officer 감사 절차를 따른다.
- `/cso`는 코드를 수정하는 기능이 아니라 보안 리포트를 만드는 읽기 전용 감사 기능이다.
- 감사는 OWASP Top 10, STRIDE 위협 모델, 의존성 공급망, 시크릿 노출, CI/CD 설정, 외부 연동, 인증/인가 경계를 우선 확인한다.
- 추측성 이슈를 많이 나열하지 말고 실제 악용 경로가 설명 가능한 항목만 주요 finding으로 보고한다.
- 각 finding에는 심각도, 신뢰도, 파일 위치, 악용 시나리오, 영향, 수정 권장사항을 포함한다.
- 기본 모드는 신뢰도 8/10 이상만 보고하는 low-noise 감사로 한다.
- 사용자가 `/cso --comprehensive`를 요청하면 낮은 신뢰도의 의심 항목도 별도 표시하되, `TENTATIVE`로 구분한다.
